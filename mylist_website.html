Craigslist-Mylist

This is an academic project I did in teams of two. In this project, we reorganized the information architecture of craigslist and rebuild the web and mobile site to make it more accessible by users.

We as a team developed the new information architecture together. Then, when executing, I am mainly responsible for creating wireframes and code for the desktop version.

The redesign process including analyzing function specifications and content requirements, iterated flow chart, wireframe, and development.

Identify Problems

By looking into the original website, we listed a few problems we are experiencing as well as interviewed other users about their experiences.

Then we concluded some prominent features we wanted to change: first, the text is too heavy, the hierarchy is not clear, some function is not seamless. Based on these three insights, I decided to use info graphic to indicate categories which clearly identified the top hierarchy.
Then we created a new structure of hierarchy for our chosen categories, housing based on the purpose of the users.

Lastly, I decided to make the posting function more obvious and seamless.

Original Craigslist site screenshot and proposed site screenshot

Flowchart and Wireframe

We first looked into the information architecture (IA) of the original website by creating different flowcharts that indicate how the information flows within the site. Then based on the problems emerged from this process as well as the analysis of the content requirements and function specifications, we consolidated our flowcharts. And of course, this goes through different phase and we concised the steps weeks by weeks based on feedbacks.

After spending a lot of time builidng the new information architecture, I started apply visual representations for these information. To do so, I present these plans using these basic wireframes and coded using basic HTML that are able to click around and get a sense of what's going on.

Coding from scratch and Responsiveness

Then we build this website from scratch. My teammate is responsible for doing the mobile version and I am responsible for doing the desktop version.
